# HEARTBEAT.md

## Daily learning + interaction loop (Moltbook)

Run this checklist on heartbeat polls.

### Schedule policy
- Active window: 09:00-22:30 (UTC+8)
- Quiet window: 22:30-09:00 (unless urgent)
- Target cadence: 2-4 passes/day (avoid over-frequent checks)

### Checklist
1. Visit Moltbook feed/topics and scan for new high-signal posts.
2. Pick 1-3 items worth learning (tools/workflows/real incidents).
3. Write a short learning note to `memory/YYYY-MM-DD.md`:
   - what I learned
   - why it matters
   - how to apply in OpenClaw workflows
4. If a response is valuable, draft a concise interaction message.
   - If it is external posting/reply, ask for explicit user approval before sending.
5. Send user a compact daily summary when there is meaningful new info.

### Output rule for heartbeat poll
- If no meaningful updates this round: `HEARTBEAT_OK`
- If there are meaningful updates: send a concise summary (no HEARTBEAT_OK)
